# 基建练卡推荐规则

> 文档角色：canonical
> 生命周期状态：current
> 领域键：advice.training
> 当前真源：self
> 摘要：定义基于玩家 Box 的确定性基建练卡建议、组合阻塞和散件筛选口径

## 产品边界

练卡推荐只回答玩家当前已经拥有的干员应培养到哪个实际基建技能门槛。规则层负责拥有状态、练度、组合完成度和优先级；自然语言或未来 RAG 只能解释结果，不能新增干员、改变目标练度或修改优先级。

- 未拥有干员不生成练卡任务；可以在体系完成度中显示缺失信息。
- 所有星级均按实际技能门槛推荐，不预设四星精一或五星、六星精二。
- 当前二档技能数据的门槛为：1-2 星 30 级、3-4 星精一、5-6 星精二；规则数据保存最终目标 `elite` / `level`，不把 `tier_up` 暴露给消费者。
- P0/P1/P2 表达当前行动优先级；核心、重要、挂件表达组合内角色，二者不能互相替代。
- 推荐只评价基建投入，不声称优先于战斗、主线或其他玩法培养。

## 组合阻塞与角色分层

组合缺少任一**必需核心**（`core` / `pick_one_core`）时，整个组合状态为缺核心或暂缓。该组合内已经拥有的核心、重要成员和挂件都不生成练卡建议；只有全部必需核心已拥有后，才按各自实际技能门槛推荐培养。

| 字段 | 组合角色 | 缺人是否阻塞 | 核心齐后练度不足 |
|---|---|---|---|
| `core` | 必需核心 | 是 | 按 `priority_ready_after_training` 生成 train |
| `pick_one_core` | 任选核心槽 | 槽内无人拥有则阻塞 | 已拥有候选未达标则 train |
| `important` | 重要成员 | 否 | 与核心同优先级 train |
| `hangers` | 挂件/外围 | 否 | 按 `priority_hangers`（默认 P2）train |

`pick_one` 核心槽有任一合法候选已拥有即满足拥有条件。同一干员仍可因独立散件价值或另一个核心完整的组合产生建议，残缺组合不能全局禁推该干员。

体系角色名单以 vault `2-体系/*.md` 成套表为准，门槛以 `MECHANICS_REGISTRY.csv` 为准；编排层非 hard required 的成员不得写进 `core`。

## 散件筛选

`data/standalone_roster.json` 是搜索和独立探测的高价值候选池，不是培养推荐表。搜索池收录只证明求解时值得保留候选，不能证明玩家值得为其投入培养资源。散件规则由维护者从该池人工筛选后写入 `data/training_recommendations.json`，且不得反向改变 solver 候选集合。

当前独立于具名组合核心的散件培养档为（门槛以 `data/MECHANICS_REGISTRY.csv` 为准）：

| 优先级 | 目标 |
|---|---|
| P0 | 石英精一、清流精一、砾精一、断罪者精一、Castle-3 30级 |
| P1 贸易 ≥30% | 慕斯/缠丸/安比尔/夜烟/芬/夜刀/古米精一；月见夜/空爆 30级 |
| P1 赤金 ≥30% | 斑点/夜烟/苍苔精一 |
| P1 经验 ≥30% | 霜叶/白雪/红豆/食铁兽精一 |
| P1 特机 | 红云精一（回收利用：仓容转生产力） |
| P2 高辨识 | 空弦/吉星/槐琥/海蒂/能天使/雪雉精二 |
| P2 贸易 30% 精二 | 可颂/拜松/衡沙/瑰盐/深巡精二 |
| P2 经验 30%+ 精二 | 裂响/酒神/弑君者/圣约送葬人/怒潮凛冬精二 |
| P2 通用 25% | 香草/史都华德/杰西卡/调香师/罗比菈塔精一（标准化·β） |

清流等条目仍按实际设施和配方结算：清流只服务赤金，收益随贸易站数量变化；这类常见设施条件必须在推荐原因中明示，但不把清流改写成自动化组硬核心。

下列情况不进入当前基础清单：

- E0 已经获得主要技能且无等级门槛时，不形成练卡任务（例如引星棘刺金属工艺·α）。
- 只有源石等非默认生产目标、稀有布局、全局资源或具名组合成立时才有推荐价值。
- 体系硬核心（但书/巫恋等）走 `system_rules`，不重复写入散件表冒充独立价值。
- 只提供心情、线索或普通补位价值，不能冒充生产推荐。

## 当前实现

- 规则载体：`data/training_recommendations.json`。
- 确定性评估：`crates/infra-core/src/training_advice/`。
- CLI 入口：`infra-cli advice --operbox <path>`。
- 人工验收稿：`python3 scripts/render_training_recommendations.py --output out/training_recommendations_review.md`。
- 公孙长乐验收流程：`.agents/skills/gongsun-training-review/SKILL.md`。
- 未拥有散件不输出；已达标散件只标记 ready；已拥有未达标散件输出 train。
- 同一干员命中多条规则时只保留最高优先级，同时合并关联体系和来源。

生成的 Markdown 只是 `training_recommendations.json` 的确定性自然语言投影，必须展示来源、冲突和待复核状态；它不是新的业务真源，也不能被手工编辑后反向覆盖结构化规则。

## 延后范围

源石等非默认配方、投入意愿、稀有布局条件、`plan --recommend-out`、前端页面和 RAG 解释层仍是独立后续工作。调查团消费方等“任意 tag 成员”暂不写入固定 hangers。
