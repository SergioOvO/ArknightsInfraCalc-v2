# 菲亚梅塔换心情规则

> 文档角色：canonical
> 生命周期状态：current
> 领域键：schedule.fiammetta
> 当前真源：self
> 摘要：裁决菲亚梅塔在当前排班中的目标和边界

> 实现快照：ABC 排班主路径已接入单次主力回岗覆盖；布局动态排序与完整心情排班尚未实现。
> 规则来源：公孙长乐 2026-07-11 评审稿。

## 当前已经实现

`plan`、`layout team-rotation` 和前端 `serve plan` 生成 ABC 排班时，会从 peak 主力选择一个菲亚梅塔目标：

```text
但书 > 巫恋 > 龙舌兰 > 清流 > 可露希尔
```

选择规则：

1. 账号必须拥有菲亚梅塔。
2. 按上述顺序检查 peak 主力，找到目标所属队伍的休息班。
3. 把目标放回 peak 原房间，并换下一个当前在岗干员。
4. 在所有合法替换位中选择该设施评分最高的方案，然后重新计算班次和全日产出。
5. 被换下者从工作 assignment 移除；具体宿舍安排由 MAA 自动执行，本项目不保证把 displaced 写入导出的宿舍槽位。
6. 只有实际发生回岗的 plan 输出 `enable: true`、`order: pre`；其余 plan 关闭。

全精二 243 当前示例中，S3 会用菲亚让但书回到贸易站，并将可露希尔换下休息。

这一步发生在基础 `A+B → B+C → C+A` rotation 之后。基础阶段的 `rotation_reserves` 负责预留并构造合法龙巫 C，但菲亚换心情允许改写最终生产房，因此最终 assignment / MAA 不保证继续保持精确 A/B/C，也不保证 C 在两个最终班次中仍是同一完整集合。

MAA 负责执行具体的宿舍拖动、心情交换和菲亚梅塔返回宿舍。InfraCalc 当前只声明是否启用、目标和使用顺序，不强制宿舍槽位。

当前每个 24 小时 αβγ 周期只安排一次回岗，因此不会再对三个 plan 逐班盲目启用。但它仍未模拟菲亚的实时剩余心情，不能视为跨多周期永续证明。

当前以公孙确认的固定目标优先级为业务规则，在合法替换位中选择瞬时评分最高者，
但不会用当前仍待校正的局部效率数值否决回岗。最终收益应由后续“主力最长工作时间
+ 替补时间 + 恢复”全周期模型判断，而不能只看单个短班的临时评分。

## 已确认的机制与策略原则

菲亚梅塔是独立的心情交换事件，不是普通宿管，也不是一支完整替补班。她的用途是让高价值主力继续工作，减少低效率替补的上班时间。

使用前提和时机：

- 菲亚梅塔本人必须满心情；
- 默认在换班前使用，即 `order = pre`；
- 使用后菲亚梅塔回宿舍恢复；
- 菲亚梅塔自身按每小时 2 点恢复，不叠加其他宿管恢复效果；
- 下一次理论就绪时间为 `(24 - 交换后菲亚梅塔心情) / 2` 小时。

后续排班层选择目标时，基础收益率建议为：

```text
菲亚收益率 = (主力效率 - 替补效率) / 主力每小时心情消耗
```

这表示主力与替补的效率差越大，菲亚梅塔的价值越高；效率差相同时，心情消耗较低、一次交换能维持更久的目标优先。

## 尚未实现的动态规则

公孙长乐给出的完整业务优先级依赖布局和成组关系：

```text
但书（二发电布局中的一级或二级贸易站）
> 巫恋 + 龙舌兰（三级贸易站）
> 但书（三级贸易站）
> 清流（三贸易站布局）
> 可露希尔
```

当前轻量 schedule 覆盖还没有布局动态排序、替补收益率、心情状态和多次菲亚使用序列，因此先采用稳定的单目标线性顺序。以下内容必须继续留在排班策略层，不能塞进 `export/maa.rs`：

- 根据布局和房间等级动态调整但书、清流的优先级；
- 将巫恋与龙舌兰作为跨班成组服务目标；
- 根据主力与替补的效率差实时计算收益率；
- 判断菲亚梅塔是否满心情、目标剩余心情和下一次就绪时间；
- 根据班次间隔、中枢心情减免和设施心情消耗决定可服务人数；
- 生成 Mower 的动态主力/替补策略，或展开成 MAA 的多个固定计划状态。

通常不建议把红松林、迷迭香等多人组合或高心情消耗的歌蕾蒂娅作为常规单点目标，但这仍属于后续排班策略，不是当前导出过滤规则。

## 代码边界

| 层级 | 当前职责 |
|------|----------|
| `schedule/` | 选择目标、执行主力回岗、换下替补并重新评分 |
| `export/maa.rs` | 把 schedule 已确认的动作写成 `enable`、`target`、`order`，并优先安排被换下者休息 |
| `infra-cli` | 调用 ABC 排班并写出 schedule 已确认的菲亚动作 |
| MAA / Mower | 执行宿舍操作与实际换班 |

本功能不等于完整宿舍恢复模拟、全基建连续心情最优化或菲亚梅塔永续证明。
