## 1. 一句话结论

自动化组是以温蒂 E2「仿生海龙」为核心、由有效电站数驱动的制造体系，但**不刚需某个最低有效电站数量**。

体系有两类制造方向：

- **赤金专用**：清流 E1 提供每贸易站 20% 的赤金生产力；典型为清流 + 温蒂 + 森蚺。
- **通用制造**：温蒂、森蚺、冬时、异客、掠风的设施计数加成不限定赤金；典型为冬时 + 温蒂 + 森蚺。

三物理电站布局（243、333、153）使用固定三人制造组。两物理电站布局（252、342）允许落在两人制造站；若同时拥有森蚺与Lancet-2，则强制采用中枢森蚺 + 发电站 Lancet-2，森蚺不进入制造站。若缺Lancet-2，森蚺回制造站。若同时拥有承曦格雷伊，则再叠加其虚拟电站。

在当前已确认锚点中：

- 243 赤金专用：清流 + 温蒂 + 森蚺，承曦格雷伊发电，140%。
- 333 赤金专用：清流 + 温蒂 + 森蚺，承曦格雷伊发电，160%。
- 153 赤金专用：清流 + 温蒂 + 森蚺，承曦格雷伊发电，120%。
- 153 通用制造：冬时 + 温蒂 + 森蚺，承曦格雷伊发电，130%，优于清流赤金组。

自动化组与莱茵生命体系**没有关系，也不共享成员**。

## 2. 领域不变量

### 2.1 激活与人数

| 类别 | 确认口径 |
|------|----------|
| 自动化核心 | 温蒂必须 E2，使用「仿生海龙」 |
| 有效电站 | 没有最低数量硬门槛；数量只影响具体效率 |
| 三电布局 | 243、333、153 的自动化制造组固定三人 |
| 两电布局 | 252、342 默认可进入两人赤金站，也可在容量允许时使用三级三人站 |
| 清流 | 最低 E1；仅赤金配方有效，不是体系关闭条件 |
| 合法通用成员 | 森蚺、冬时、异客、掠风；三电布局的第三人顺序为森蚺 ＞ 冬时 ＞ 异客 = 掠风 |

| 无赤金线 | 不关闭体系；可改走通用制造组合 |

### 2.3 配方作用域

| 干员 | 作用域 |
|------|--------|
| 清流 | 只对赤金配方生效：每贸易站 +20% |
| 温蒂 | 通用制造：每有效电站 +15%，并清零同行普通技能面 |
| 森蚺 | 制造时为通用制造：每有效电站 +10%；252/342 有森蚺时改驻中枢 |
| 冬时 | 通用制造：按本房人数提供生产力，三级三人站为 30%，两人站为 20% |
| 异客 / 掠风 | 通用制造：每有效电站 +5% |
| 承曦格雷伊 | 发电站产生 VirtualPower +1，受“其他发电站无作业平台”条件约束 |
| 中枢森蚺 + Lancet-2 | 森蚺在中枢且 Lancet-2 在发电站时产生 VirtualPower +2 |

252/342最高效率两人赤金站还有额外状态要求：Lancet-2 必须以0心情红脸状态上班。该状态下仍满足中枢森蚺联动，但不应阻断承曦格雷伊的「晨曦」。下班后 Lancet-2 放入加工站，不进入宿舍恢复。

## 3. 成套组成

### 3.1 三电布局的两类组合

```text
赤金专用：清流 E1 + 温蒂 E2 + 森蚺 E2
  清流：20% × 贸易站数，仅赤金
  温蒂：15% × 有效电站数，通用
  森蚺：10% × 有效电站数，通用

通用制造：冬时 + 温蒂 E2 + 森蚺 E2
  冬时：三级三人站 30%，通用
  温蒂：15% × 有效电站数，通用
  森蚺：10% × 有效电站数，通用
```

三电布局不能机械地固定清流。应根据贸易站数量、目标配方和组合总效率，在赤金专用组与通用制造组之间选择。153 只有一个贸易站，清流仅提供 20% 赤金效率，因此冬时 + 温蒂 + 森蚺的 130% 通用组优于清流组的 120%。

### 3.2 成员角色

| 角色 | 干员 | 设施 | 最低练度 | 说明 |
|------|------|------|----------|------|
| 自动化核心 | 温蒂 | 制造 | E2 | 仿生海龙；通用制造效率 |
| 赤金专用候选 | 清流 | 制造 | E1 | 每贸易站 +20%，仅赤金 |
| 通用首选 | 森蚺 | 制造或中枢 | E2 | 三电布局优先制造；252/342有Lancet-2时强制中枢，缺Lancet-2时回制造 |
| 通用候选 | 冬时 | 制造 | 解锁流程优化的练度 | 按房间人数提供通用效率 |
| 通用候选 | 异客 / 掠风 | 制造 | 解锁自动化 α 的练度 | 二者同级，每有效电站 +5% |
| 虚拟电源 | 承曦格雷伊 | 发电站 | E2 | 条件满足时 VirtualPower +1 |
| 两电布局 producer | 森蚺 | 中枢 | E2 | 与 Lancet-2 联动，VirtualPower +2 |
| 两电布局挂件 | Lancet-2 | 发电站 | 0 | 中枢森蚺联动条件 |

### 3.3 243 不使用中枢森蚺

243 有三物理电站，森蚺留在制造站可直接提供每有效电站 10% 的通用制造效率。将森蚺移入中枢并占用 Lancet-2 发电位，会损失制造第三人和其他中枢/发电收益，因此默认不采用该分支。

### 3.4 252/342 的中枢森蚺分支

252/342 只有两物理电站：

- 同时拥有森蚺与Lancet-2时，强制森蚺进中枢、Lancet-2 进发电站，VirtualPower +2。
- 森蚺进入中枢后，禁止同时进入制造站。
- 同时拥有承曦格雷伊时，再安排其进另一发电站，额外 VirtualPower +1。
- 无承曦格雷伊时不关闭体系，只按较低有效电站数结算。
- 默认两人赤金站为清流 + 温蒂；无温蒂 E2 时不启用该自动化组合。
- Lancet-2 必须0心情红脸上班；该组下班时转入加工站，不进宿舍。
- 三级三人站中森蚺已进中枢时，第三位按冬时 ＞ 异客 = 掠风选择。
- 有森蚺但没有Lancet-2时，不走中枢联动，森蚺回制造站。

## 4. 班次与房间约束

- 自动化组合工作两个班次、休息一个班次。
- 第一班默认是主力班；第二个工作班可以是第二班，也可以是第三班，只要符合正常三队换班逻辑。
- 制造组与所有为该组提供虚拟电站的 producer 必须绑定到同一队、同上同下。
- 三电布局绑定制造三人 + 承曦格雷伊（若使用）。
- 252/342 绑定制造成员 + 中枢森蚺 + Lancet-2 + 承曦格雷伊（若使用）。
- Lancet-2 随该组停止发电工作时进入加工站，不进入宿舍；不能套用普通休息队宿舍分配。
- 不固定队名、房间 id 或休息班次编号。
- 清流只允许进入赤金配方；通用制造组可进入合法制造配方。

## 5. 数值锚点

“精确数值锚点”是指：给定布局、配方、成员和 producer 后，明确写出每个分量及总效率，用于验证程序没有靠错误常量或错误配方得到相同总数。

### 5.1 已确认锚点

以下均假设三物理电站 + 承曦格雷伊，即有效电站数为 4。

| 配置 | 布局 | 配方 | 清流 | 冬时 | 温蒂 | 森蚺 | 合计 |
|------|------|------|------|------|------|------|------|
| 清流 + 温蒂 + 森蚺 | 243 | 赤金 | 2贸 × 20% = 40% | — | 4电 × 15% = 60% | 4电 × 10% = 40% | 140% |
| 清流 + 温蒂 + 森蚺 | 333 | 赤金 | 3贸 × 20% = 60% | — | 60% | 40% | 160% |
| 清流 + 温蒂 + 森蚺 | 153 | 赤金 | 1贸 × 20% = 20% | — | 60% | 40% | 120% |
| 冬时 + 温蒂 + 森蚺 | 153 | 通用 | — | 三人站 30% | 60% | 40% | 130% |

冬时 + 温蒂 + 森蚺在同样的四有效电站条件下，对243/333也为130%通用制造效率；但243/333的赤金清流组分别为140%/160%，因此赤金线上优先清流组。

### 5.2 无承曦格雷伊不是关闭档

无承曦格雷伊时，三电布局按三个有效电站重新结算，不再使用旧文档“档3约90%”口径。

旧档3写的是清流 + 温蒂 + 冬时、三个物理电站。按同一公式应为：

```text
243：清流 40% + 温蒂 3×15%=45% + 冬时三人站 30% = 115%
```

因此旧“约90%”是错误锚点；该状态只是无虚拟电站的正常降级，不是关闭条件，也不单独绑定 priority 14。

## 7. 关闭与降级语义

1. 无温蒂 E2：不形成以仿生海龙为核心的自动化组，退回普通制造组合。
2. 无清流或清流未到 E1：不关闭，不能使用清流赤金分支，改评估通用制造组合。
3. 无承曦格雷伊：不关闭，按实际有效电站数降低效率。
4. 有效电站数量没有关闭阈值；不得再使用“至少4”或“低于3关闭”的旧规则。
5. 三电布局的完整自动化制造组固定三人；缺合法第三人时不声称形成满配组，但体系也不整体关闭。
6. 两电布局允许两人制造站；有森蚺时必须切换到中枢森蚺 + Lancet-2 分支。
7. 无赤金线时不关闭，可使用不含清流的通用制造组合。
8. 两电布局的清流 + 温蒂组合缺温蒂 E2时不启用。
9. 三电布局缺森蚺时，以温蒂为核心，从冬时 ＞ 异客 = 掠风中选两名同伴；凑不满三名合法成员时，整房改走普通制造组合。
10. 两电布局有森蚺但没有Lancet-2时，森蚺回制造站，不强行占用中枢。
11. 两电布局清流 + 温蒂两人赤金站在2、3、4、5有效电站时均可正常启用。
12. 两电两人通用站按实际效率动态排序；三级三人通用站使用 §5.5 的固定锚点。
13. 有森蚺但无Lancet-2时，森蚺固定回三级制造站，并使用 §5.5 的赤金/通用锚点。

## 9. 当前实现审计结论（2026-07-12，只读）

### 9.1 已正确建模

- 清流数据为每贸易站 +20%，仅赤金；最低练度应为 E1。
- 温蒂仿生海龙为每有效电站 +15%。
- 森蚺制造技能为每有效电站 +10%。
- 冬时按房间人数提供通用制造效率。
- 异客/掠风自动化 α 为每有效电站 +5%。
- 承曦格雷伊可以产生 VirtualPower +1。
- 中枢森蚺 + 发电站 Lancet-2 可以产生 VirtualPower +2。
- 标准全精二样例当前得到：153清流组120%、243清流组140%、333清流组160%。
- 当前MAA导出会忠实复制core排班；导出层不是已发现问题的根因。

### 9.2 未实现或错误实现

- 当前选型把清流固定为自动化组成员，没有实现“清流赤金组 vs 通用制造组”的布局/配方比较，因此153没有选择130%的冬时 + 温蒂 + 森蚺。
- 当前252/342没有安排中枢森蚺 + Lancet-2；实际仍把森蚺放在制造站。
- 当前真实252输出115%、342输出135%，只按两个物理电站 + 承曦格雷伊计算，未达到确认后的producer语义。
- 当前结果虽然数值碰巧等于新确认的115%/135%，但成员与资源来源错误：代码使用制造森蚺贡献，目标口径要求中枢森蚺 + Lancet-2，并且制造站只有清流 + 温蒂。
- `template_252_auto1` 对应文件不存在；缺失layout被fallback成标准243 fixture，旧252单元测试实际使用了三个物理电站。
- 当前第三人专用补位只包含森蚺、冬时，没有异客 = 掠风分支。
- 自动化组没有结构化 `shift_bind`；标准样例第一/三班工作只是当前房间顺序和切半顺序的结果。
- 专用制造fill只检查第一间赤金房；双anchor位于后续赤金房时缺少结构化保留证明。
- 旧完整resolve回归把自动化三人放进战斗记录房，却按清流赤金公式断言130%，fixture与公式冲突。
- 当前排班没有为单个Lancet-2表达0心情红脸工作状态；作业平台判定也没有该状态语义，无法证明其与承曦格雷伊晨曦同时生效。
- 当前休息队导出会把未工作成员送入宿舍，未实现Lancet-2下班进入加工站且禁止进宿舍的规则。
- 当前实现没有有效电站阈值门禁；这一点与新口径一致，不应再新增阈值关闭规则。
- 当前实现与莱茵生命没有体系关系；新口径确认无需建立该关系。

### 9.3 建议的单一责任边界（不实施代码）

- 体系选型应唯一决定三电/两电分支、制造人数、配方方向和producer集合。
- 选中的制造成员与producer应作为required成员进入统一计划；bind不能代替进编保证。
- 制造fill只负责在合法容量与配方中保留全部已选成员，不应再次按中文名重判体系。
- resolve继续按通用EffectAtom结算，不写布局名、房号或体系名特判。
- rotation只消费已激活体系产出的同队绑定，不重新决定成员。
- export继续忠实复制core assignment，不承担修复职责。
