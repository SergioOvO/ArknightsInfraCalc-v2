## 目标

让 current truth、active change、ADR、generated facts、evidence 和 archive 具有唯一且可机械证明的生命周期边界；完成存量迁移后阻止重新漂移。

## 不变量

- 每个受治理 Markdown 恰好一个角色和合法状态。
- 每个领域键恰好一个 canonical owner，且 base/head 连续。
- `docs/TODO/` active-only，根 `plans/` 不保留 Markdown。
- 关闭项完成事实吸收、开放项拆分、移动/删除、索引引用和证据事务。
- 文件数不是停止条件；只有语义冲突、产品范围或用户裁决冲突才询问。

## 用户裁决

- 同时处理存量迁移和长期门禁；完成后零豁免 hard fail。
- 纯重复稿删除，有独立决策或审计价值的材料归档。
- 排班收敛为“模式总纲 + 定时细则”；`SCHEDULE_ROTATION.md` 只记录当前 ABC 实现。
- 四份体系文档分别作为 canonical；`SYSTEM_CHAINS.md` 收敛为导航，`new/* (1).md` 归档为历史审计。
- 红松旧审计附录 A 不覆盖当前迷迭香文档。
- 红松林赤金线不是激活前提；旧“三制造/至少一赤金”前提删除。
- 文档/代码差异只有涉及业务语义时询问；实现状态按代码和 evidence 更新。
- 本迁移不修 Rust/领域行为；发现实现缺陷拆独立任务。
