## 1. 现状与问题

### 当前 ABA 排班（`schedule/base_rotation.rs`）

| 班次 | 角色 | 说明 |
|------|------|------|
| Shift 1 (A) | **Peak** 高峰班 | 贸易 meta 全出（但书/龙巫/可露希尔） |
| Shift 2 (B) | **Recovery** 恢复班 | 孑精0带队 + 余量贪心 |
| Shift 3 (A') | **Peak** 复用 shift1 | 同一批人再次上岗 |

**问题**（公孙长乐指正）：
- ABABAB 这种换班方法，**需要为每个站多准备一组人** → 干员利用率降低
- **宿舍一次性休息不下** → 大量干员同时下班，宿舍床位不够
- 但书/黑键等关键干员可能连上 3 班无休（如 `schedule_export.json` 实际数据所示）
- 中枢/宿舍钉死，无法随队调整

## 2. 新设计：αβγ 三队均衡轮休

### 2.1 核心原则

1. **按站分组**：贸易站 α、β、γ 各绑定一支独立团队
2. **工作 18h / 休息 6h**：每团队工作 2 个班次、休息 1 个班次
3. **12h + 6h + 6h 班次结构**：效率最高的两队上 12h 班，6h 班为两队轮流休息时间（公孙长乐关键指正）
4. **心情自洽**：见 §5

### 2.2 三班轮换矩阵（修正版）

```
           12h班次(主力)   6h班次        6h班次
α 队        上班            休息          上班    ← 工作18h 休息6h
β 队        上班            上班          休息    ← 工作18h 休息6h
γ 队        休息            上班          上班    ← 工作18h 休息6h
```

| 班次 | 时长 | 特点 |
|------|------|------|
| 班次 1（长班） | **12h** | **最高效两队 α+β 同时上岗**，γ 队休息 |
| 班次 2（短班） | **6h** | β 队继续上岗 + γ 队接班，α 队休息 |
| 班次 3（短班） | **6h** | γ 队继续上岗 + α 队接班，β 队休息 |

**每队节奏**：工作 12h + 6h = 18h，休息 6h → 24h 循环

## 3. 各设施岗位分配

### 3.1 243 蓝图岗位矩阵（修正版）

| 设施 | 站数 × 每站人 | 12h班上岗 | 6h班上岗 | 轮转池 |
|------|--------------|----------|---------|-------|
| 贸易站 | 3 × 3 = 9 | **6** (α+β) | **6** (β+γ 或 γ+α) | 9人 |
| 制造站 | 4 × 3 = 12 | **9** (3站) | **9** (3站) | 12人 |
| 发电站 | 3 × 1 = 3 | **3** (全上) | **3** (全上) | 5人(3+2共用替补) |
| **轮转合计** | | **18** | **18** | **~26** |
| 中枢 | 1 × 5 | 5 | 最多5人轮休 | 随体系绑定 |
| 宿舍 producer | 4 × 1 | ~4 | ~4 | ~4 |

### 3.2 发电站轮换方案（公孙长乐指正）

发电站不按 αβγ 三队分，而是三个站 2 个一组共用替补：

```
发电站 A + 发电站 B → 共用替补 X（3 人轮换 2 岗）
发电站 C 独立轮换 → 替补 Y（2 人轮换 1 岗）
总计：3 + 2 = 5 人即可
```

每班 3 个发电站全部运转，不空转。

### 3.3 αβγ 三队构成

| 队 | 贸易 | 制造 | 发电 | 中枢(体系) | 宿舍 |
|----|------|------|------|-----------|------|
| α 队 | trade_1 | 2站 | 共替补 | 八幡海铃(但书链)等 | ∼1 |
| β 队 | trade_2 | 1站 | 共替补 | 灵知(喀兰)等 | ∼1 |
| γ 队 | trade_3 | 1站 | 共替补 | 火龙S黑角(木天蓼)等 | ∼1 |

## 4. 控制中枢策略

**决策**：体系绑定干员跟贸易站团队走，其余位随意轮换（已确认）。

| 中枢绑定体系 | 绑定干员 | 跟队策略 |
|-------------|---------|---------|
| 叙拉古·但书链 | **八幡海铃** E2 | 跟 α 队（但书所在队） |
| 木天蓼·调查团 | **火龙S黑角+麒麟R夜刀** | 跟 γ 队（需 consumer 同班） |
| 人间烟火 + 感知信息 | **夕/令** 等 | 通用，任意班次 |
| 喀兰贸易 | **灵知** | 跟 β 队（待定） |
| 推王摩根 | **戴菲恩** | 跟 α 或 γ 队（待定） |
| 心情回复补位 | **薇薇安娜/焰尾** 等 | 通用补位 |
| 数值加成位 | +7% 贸易 / +2% 制造 等 | 随队轮换 |

**中枢轮休**（公孙长乐指正）：中枢每班最多 5 人休息，可轮换。

## 5. 心情回复计算（修正版）

**实际游戏数据**（公孙长乐指正）：
- 满级宿舍（Lv5）心情回复：**4 点/小时**
- 中枢 / 发电 / 人力 / 会客 心情消耗：**0.75 点/小时**
- 三级制造站 / 三级贸易站 心情消耗：**0.65 点/小时**

### 12h 班次（最长连续上岗）

| 岗位类型 | 消耗/h | 12h 消耗 | 6h 休息回复 | 净变化 |
|---------|--------|---------|------------|--------|
| 贸易站 Lv3 | 0.65 | 7.8 | 6h × 4 = 24 | **+16.2** |
| 制造站 Lv3 | 0.65 | 7.8 | 6h × 4 = 24 | **+16.2** |
| 中枢 | 0.75 | 9.0 | 6h × 4 = 24 | **+15.0** |

### 18h 循环总消耗（连续 12h + 6h 两班）

| 岗位 | 12h 消耗 | 6h 消耗 | 总消耗 | 6h 休息回复 | 净变化 |
|------|---------|--------|--------|------------|--------|
| 贸易站 Lv3 | 7.8 | 3.9 | 11.7 | 24 | **+12.3** ✅ |
| 中枢 | 9.0 | 4.5 | 13.5 | 24 | **+10.5** ✅ |

**结论**：即使连续上 18h 班（12h+6h），消耗也远小于 6h 休息的回复（24 点）。**心情完全自洽，无需菲亚梅塔。**

## 7. 实现方案

### 7.1 新增数据结构

```rust
/// αβγ 三队标签
enum TeamLabel { Alpha, Beta, Gamma }

/// 班次时长
enum ShiftDuration { Hours12, Hours6 }

/// 三团队轮换排班
struct TeamRotationPlan {
    teams: Vec<TeamPlan>,              // [α队, β队, γ队]
    shift_schedule: Vec<ShiftPlan>,    // 3个班次
}

struct TeamPlan {
    label: TeamLabel,
    trade_station_idx: usize,          // 绑定某贸易站
    manufacture_station_idxs: Vec<usize>, // 绑定的制造站
    operators: Vec<String>,            // 该队所有上岗干员
    system_bound_control: Vec<String>, // 绑定的中枢体系干员
}

struct TeamShiftSlot {
    station_idx: usize,
    operators: Vec<AssignedOperator>,
    score: f64,
}

struct ShiftPlan {
    index: usize,                      // 0, 1, 2
    duration_hours: f64,               // 12.0 or 6.0
    active_teams: Vec<TeamLabel>,      // 这班哪些队上岗
    resting_team: TeamLabel,           // 这班休息的队
    assignment: BaseAssignment,        // 全基建编制
    scores: ShiftScores,
}
```

### 7.2 搜索与指派流程

```
输入: blueprint + operbox + instances + table
         ↓
  1. 构建全设施 pool（trade / manu / power / control）
         ↓
  2. 分三队搜索，各队内部 used 剪枝:
     贸易站 × 3 → α/β/γ 各绑一站
     制造站 × 4 → α=2站 β=1站 γ=1站
     发电站 × 3 → 3+2=5人轮换方案(独立)
     中枢 → 体系绑定跟队, 其余贪心
     宿舍 → 跟队分配
         ↓
  3. 生成 3 个班次的 BaseAssignment:
     班次0(12h):  α上岗 β上岗 γ休息
     班次1(6h):   α休息 β上岗 γ上岗
     班次2(6h):   α上岗 β休息 γ上岗
         ↓
  4. 对各班次 resolve_base + 评分
         ↓
  5. 输出 TeamRotationReport
```
