# 罗德岛基建管家



# 一、预期目标

1. 预期成果：为玩家提供一站式基建规划和指引的工具，把基建交给管家，让玩家都可以当上甩手掌柜

2. 功能模块：
①用户注册与账号管理（网页）
②干员box和练度读取比对
③干员组合情况与培养建议
④需求诊断与设计排班表方案
⑤导出作业至maa/mower等工具
⑥基建技能列表与筛选查询
⑦报表分析与健康诊断
⑧\<远期\>理论最优基建方案计算

3. 成果形式：
常态更新可用的网页服务（高互动），和支持部署到本地的应用（低维护成本）

# 二、模块一：干员box和练度读取比对

1. 支持从maa的练度读取结果，或者森空岛api两个途径来获取干员的数量和练度；maa练度读取的为文件上传解析；森空岛api为api解析

2. 与基建技能解锁的精英化等级、基建组合表单进行比对，获取干员box特征信息

3. 对于基建幻神组，或者常用三星四星精一/赠送干员（断罪者、苍苔等），高亮提示缺少或者推荐，输出建议练卡或者收集建议

4. 对于可以在同一个组合内上班的人，进行分组展示，提供上下位关系

# 三、模块二：需求诊断和排班表方案设计

1. 通过问卷选项或者自然语言交流的方式，进行玩家的需求诊断。
诊断内容包括：①基建目标是搓玉/平衡产出/极大化钱/极大化经验；②是否有意愿为了基建产能提升而投资干员，倾向是完全不愿意仅使用三四星或者强卡六星角色/愿意投资部分高性价比组合/接受为了提升哪怕1%的效率练卡；③日常游玩习惯，新人还是老玩家，氪金情况、上线频次与间隔、摆烂自动轮换还是使用手动队列轮换、是否有满级强迫症等等；

2. 基于干员box可达成情况，和采集到的玩家需求倾向，制定适合玩家的基建排班表，以图片或者组合形式展示。允许玩家对该内容进行导出
对于尚未达到练度要求的干员进行列示，建议进行收集或者培养

3. 给出预计的产出情况，并支持在完善参数后导出该作业成maa文件

4. \<远期\>提供一个基于mower跑单的方案自动设计，导出成mower的作业。但mower的逻辑自成体系，比正常换班的复杂很多，之后再议

# 四、模块三：AI问答助手

炼一个公孙长乐\.skill，支持问答和交流。比如对上述需求的诊断；询问某个干员的技能，缺少某个组合或者干员如何进行替换或者搭配，产出的计算或者是否正常；甚至锐评一下基建强度（

提供skill的开源部署，让大家可以在自己电脑上烧自己token

在线的问答肯定要烧token，目前想把网页上的该功能作为一个允许全体玩家每天有限次数提问，给网站打赏/充月卡的玩家可无限自由提问，用收益来维持token消耗、网页流量和后续开发支出。目标主要是为了让项目能够实现自负盈亏，不要成为黑洞

# 六、模块五：报表分析与健康诊断

1. 支持玩家上传三日报表或填写三日报表，对其平均数值进行健康程度诊断，是否存在产出明显偏低，钱书比如何，订单质量与赤金平衡等情况；

2. 支持玩家输入或者上传库存钱书，计算钱书缺口，以明确目标倾向

3. 支持对365年度工作总结报告的基建产出情况进行分析（然而今年的年度总结没出）

4. \<远期\>给玩家提供长期记录选项，可以记个一个月、一年的财报情况，并丢给ai助手进行分析或者总结汇报
