## 1. 迷迭香感知链

### 一句话

迷迭香（制造）+ 黑键（贸易）绑定同上同下，以感知信息（Perception）为核心中间件，搭配感知 producer 堆高全局感知，经思维链环/无声共鸣转化为制造效率和贸易效率。243/153/333 三电布局下满配约 **80–90% 制造 / 40–45% 贸易**。

### 硬性前提

| 条件 | 说明 |
|------|------|
| 布局 | 三发电（243/153/333）最佳；二发电可选但产出缩水 |
| 迷迭香 | **必须 E2** |
| 黑键 | **必须 E2**，与迷迭香同上同下 |
| 絮雨 | **推荐 E2**（办公室三级，+20 感知）；缺位时替代感知源 |

### 感知 producer

| 干员 | 设施 | 产出 | 优先级 |
|------|------|------|--------|
| 夕 | 中枢 | E0 即可，心情>12 产感知 10；<12 转人间烟火 | 推荐 |
| 絮雨 | 办公室三级 | 记忆碎片，感知 +20 | **推荐** |
| 爱丽丝 | 宿舍 | 梦境呓语，每宿舍等级 +1 感知 | 推荐 |
| 车尔尼 | 宿舍 | 琴键漫步，每宿舍等级 +1 感知 | 推荐 |
| 琴柳 | 中枢 | 与絮雨搭配，补齐感知约 20 | 可选项 |
| 令 | 中枢 | 替代夕，需 ≤12 心情 | 可选项 |

### 效率锚点

| 配置 | 迷迭香制造 | 黑键贸易 |
|------|-----------|---------|
| 三核心 + 夕 + 爱丽丝 | **75%** | **37.5%** |
| + 絮雨 + 琴柳 + 令(>12) | **80%** | **40%** |
| + 絮雨 + 琴柳 + 令(≤12) | **90%** | **45%** |

### 代码状态

| 组件 | 状态 |
|------|------|
| 感知 producer / consumer（`global_resource/`） | ✅ 已实现 |
| `base_systems.json` 条目 | ✅ 完整 `rosemary_perception` 不进编；黑键可露只保留为 L3/segment 锚点 |
| 但书站 + 黑键搭配 | ✅ role policy：但书站优先，黑键通过 `closure` role 优先服务可露希尔站 |
| 夕 elite 层级（应为 0） | ⚠️ 需确认 |

### 约束

- 黑键**不与巫恋同站**
- 迷迭香+黑键**上 2 班休息 1 班**
- 贸易站按但书 → 可露希尔 → 巫恋 core priority。243 full E2 时黑键常作为可露希尔站高效工具人；缺黑键时可露仍上站，龙巫作为第三优先级走裁缝 β / α / 空白 fallback。
- 恢复班不启用成套

## 2. 自动化组

### 一句话

利用虚拟电站（承曦格雷伊 ± 森蚺中枢+Lancet-2）堆高有效电站数，驱动温蒂「仿生海龙」清零同行 + 每电站 15% 加成，配合清流跨设施 20%/贸易站。243 典型配置 **140%**（清流+温蒂+森蚺）。

### 硬性前提

| 条件 | 说明 |
|------|------|
| 温蒂 | **必须 E2** |
| 有效电站 ≥ 4 | 自动化收益 = 电站数 × 15% |
| 产物 | **赤金专用** |
| 第三人首选 | 森蚺 E2 > 冬时 E2 > 异客/掠风 |

### 效率锚点

| 配置 | 布局 | 有效电站 | 合计效率 |
|------|------|---------|---------|
| 清流+温蒂+森蚺 | 243 | 4 | **140%** |
| 清流+温蒂+冬时 | 243 | 4 | **130%** |
| 清流+温蒂+森蚺 | 333 | 4 | **160%** |
| 清流+温蒂+森蚺 | 153 | 4 | **120%** |

### 代码状态

| 组件 | 状态 |
|------|------|
| 温蒂·仿生海龙 | ✅ 已实现 |
| 清流 buff（+20%/贸）| ⚠️ 需核实数值 |
| 森蚺·自动化 β | ✅ 已实现 |
| 承曦格雷伊·VirtualPower +1 | ✅ 已实现 |
| `base_systems.json` 条目 | ❌ 不存在 |

### 约束

- 与迷迭香感知链**完全互补**（各占一条赤金线）
- 与红松林**完全正交**
- 第一/三班启用，第二班休息
- 无温蒂 E2 或有效电站 < 3 → 不开

### 缺人降级路径

```
满配 140%（温蒂+清流+森蚺+承曦）
  ├─ 冬时替补 130%
  ├─ 异客替补 ~110%
  └─ 缺温蒂 E2 → 关闭
```

## 3. 红松林体系

### 一句话

焰尾 E2 + 薇薇安娜 E2 双中枢，为灰毫/远牙/野鬃（经验线）和砾（赤金线）提供跨设施制造增益。满配经验房 **126%**、赤金房 **42%**，净提升 **+20% 经验 +7% 赤金**。

### 硬性前提

| 条件 | 说明 |
|------|------|
| 焰尾 | **必须 E2** |
| 薇薇安娜 | **必须 E2**（无她时系统净提升仅 0–10%，不值得开）|
| 红松林制造干员 | 至少 2 人（灰毫/远牙/野鬃），E2 为佳 |
| 布局 | 三制造站以上，至少 1 经验 + 1 赤金 |

### 效率锚点

**经验房（灰毫+远牙+野鬃）：**
| 来源 | 每人 | 房间合计 |
|------|------|---------|
| 红松骑士团·β 本体 | 25% | 75% |
| 焰尾 buff（2 同伴 ×10%）| 20% | 60% |
| 薇薇安娜 buff | 7% | 21% |
| **合计** | **52%** | **126%** |

**赤金房砾：42%**（本体 35% + 薇薇安娜 7%）

### 代码状态

| 组件 | 状态 |
|------|------|
| 灰毫/远牙/野鬃 制造 L1 atom | ✅ 已完成 |
| 焰尾 E2「红松的骑士」| ✅ `control_mp_psk[000]` + `cc.g.pinus` tag |
| 薇薇安娜 E2 制造 buff | ✅ `control_prod_fraction[000]` + 砾 `cc.g.knight` |
| `base_systems.json` 条目 | ✅ `pinus_sylvestris`（priority 19） |

### 约束

- 第一/三班启用，第二班休息
- 无焰尾 E2 或无薇薇安娜 E2 → 不开
- 恢复班不启用

### 缺人降级

```
满配（三红松林+砾+焰尾+薇薇安娜）126%/42%
  └─ 缺 1 名红松林 → 替补 35% 经验散件（食铁兽等）
  └─ 无薇薇安娜 → 净提升 0-10%，不推荐
```

## 4. 莱茵生命体系

### 一句话

包含**两个独立子系统**：

- **莱茵科技技能组合**（同房搭配）：多萝西（每莱茵科技技能 +5%）+ 溯光星源（每技能 +5 容量）+ 莱茵科技 α/β/γ 持有者
- **莱茵生命阵营增益**（全基建被动）：缪尔赛思（电站，每人 +3%）和娜斯提（赤金，每人 +3%）

### 效率锚点

**莱茵科技组合（同房）：**
| 组合 | 估算效率 |
|------|---------|
| 多萝西+淬羽赫默（2 技能）| ~70% |
| +娜斯提（3 技能）| ~105% |
| +星源/白面鸮（4 技能）| ~140%+ |

**莱茵阵营增益（被动）：**
| 莱茵在基建人数 | 娜斯提赤金加成 | 缪尔赛思电站加成 |
|---------------|--------------|-----------------|
| 5 人 | **+15%** | **+25%** |

### 代码状态

| 组件 | 状态 |
|------|------|
| 莱茵科技 α/β/γ 本体 | ✅ 已实现 |
| 多萝西 per-skill +5% | ✅ 已实现 |
| 溯光星源 per-skill +5 容量 | ✅ 已实现 |
| 缪尔赛思·基础 +10% | ✅ 已实现 |
| 缪尔赛思·per-Rhine +3% | ✅ `power_rec_rhine[000]` + `cc.g.rhine` tag |
| 娜斯提·造价高昂 | ✅ `manu_formula_spd&cost_bd[100]` + `RhineLifeInBase` |
| `cc.g.rhine` tag | ✅ 10 干员 × tier |

### 约束

- 莱茵科技组合采用**半固定**：多萝西 + 淬羽赫默 核心固定，第三人靠 C(n,3) 贪心
- 莱茵阵营增益**不绑定生产设施**（宿舍/会客等也算）
- 不需要 `base_systems` 条目（C(n,3) 贪心自然选出高分组合）
