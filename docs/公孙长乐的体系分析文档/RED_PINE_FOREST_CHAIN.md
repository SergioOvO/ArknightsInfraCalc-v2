## 1. 一句话结论

红松林体系是**中枢→制造跨设施作战记录专精体系**，以焰尾 E2 + 薇薇安娜 E2 双中枢为核心，为灰毫/远牙/野鬃（经验线）和砾（赤金线）提供跨设施制造增益。满配（含薇薇安娜）经验房 **126%**、赤金房 **42%**，净提升 **+20% 经验 +7% 赤金**。priority 19，略低于但书 20。

## 2. 何时值得开（决策表：布局 × 盒内干员 × 已有 meta）

### 2.1 硬性前提

| 条件 | 说明 |
|------|------|
| 焰尾 | **必须 E2** |
| 薇薇安娜 | **必须 E2**（无薇薇安娜时系统净提升仅 0–10%，不值得开）|
| 灰毫/远牙/野鬃 | 至少 2 人，E2 为佳 |
| 产物 | **作战记录**（焰尾 buff 对赤金为惩罚）|
| 布局 | 三制造站以上，至少 1 间作战记录 + 1 间赤金 |

### 2.2 不开的情况

- **无焰尾 E2 或无薇薇安娜 E2** → 不开（净提升不足）
- **红松林制造干员不足 2 人** → 不开
- **该班次无极作战记录产线** → 不开
- **中枢 5 人已满且焰尾/薇薇安娜都不可替换** → 不强行插入
- **恢复班** → 不启用成套

### 2.3 与已有 meta 的共存

| 体系 | 关系 |
|------|------|
| 迷迭香感知链 | **完全兼容**。迷迭香占赤金线，红松林占经验线；中枢 5 人位充裕 |
| 但书组（叙拉古）| **完全正交**。不影响贸易 |
| 龙门中枢（斩业星熊+诗怀雅）| **兼容**。中枢 5 人位 |
| 怪猎中枢 | **兼容**。中枢 5 人，焰尾+薇薇安娜+怪猎双人 = 4 位，余 1 位 |
| 喀兰中枢（灵知）| **兼容**。中枢内不互斥（红松林不入 `meta_chain`）|
| 人间烟火链（乌有+令+重岳）| **兼容** |

## 9. 建议落地的代码改法（条目化）

### 9.1 红松林相关 buff 建模

**焰尾 E2「红松的骑士」（`control_mp_psk[000]`）**：
- 当前 `atoms: []`，需要实现跨设施条件增益
- 建议方案：复用 `control` 注入管线，
  - 统计制造站 `cc.g.pinecone` tag 干员数（当前可以确认灰毫/远牙/野鬃有此 tag，砾可能没有）
  - 向 `GlobalInjectManifest` 写入红松林计数
  - 制造站 interpreter 在计算效率时读取该计数，作用给持有 `cc.g.pinecone` tag 的干员本身

- 需要确认其 `buff_id` 和当前建模状态

## 附录 A：迷迭香体系补充确认（基于公孙回答的修正）

- **所有布局默认办公室三级**，蓝图中必然有办公室。

- 代码里判断：琴柳是中枢填充位的可选项，不影响感知链主体。
