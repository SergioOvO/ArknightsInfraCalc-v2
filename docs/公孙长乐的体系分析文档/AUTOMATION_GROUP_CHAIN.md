## 1. 一句话结论

自动化组是以温蒂 E2「仿生海龙」为核心、由有效电站数驱动的制造体系，但**不刚需某个最低有效电站数量**。

体系有两类制造方向：

- **赤金专用**：清流 E1 提供每贸易站 20% 的赤金生产力；典型为清流 + 温蒂 + 森蚺。
- **通用制造**：温蒂、森蚺、冬时、异客、掠风的设施计数加成不限定赤金；典型为冬时 + 温蒂 + 森蚺。

三物理电站布局（243、333、153）使用固定三人制造组。两物理电站布局（252、342）允许落在两人制造站；若同时拥有森蚺与Lancet-2，则强制采用中枢森蚺 + 发电站 Lancet-2，森蚺不进入制造站。若缺Lancet-2，森蚺回制造站。若同时拥有承曦格雷伊，则再叠加其虚拟电站。

在当前已确认锚点中：

- 243 赤金专用：清流 + 温蒂 + 森蚺，承曦格雷伊发电，140%。
- 333 赤金专用：清流 + 温蒂 + 森蚺，承曦格雷伊发电，160%。
- 153 赤金专用：清流 + 温蒂 + 森蚺，承曦格雷伊发电，120%。
- 153 通用制造：冬时 + 温蒂 + 森蚺，承曦格雷伊发电，130%，优于清流赤金组。

自动化组与莱茵生命体系**没有关系，也不共享成员**。

## 3. 成套组成

### 3.1 三电布局的两类组合

```text
赤金专用：清流 E1 + 温蒂 E2 + 森蚺 E2
  清流：20% × 贸易站数，仅赤金
  温蒂：15% × 有效电站数，通用
  森蚺：10% × 有效电站数，通用

通用制造：冬时 + 温蒂 E2 + 森蚺 E2
  冬时：三级三人站 30%，通用
  温蒂：15% × 有效电站数，通用
  森蚺：10% × 有效电站数，通用
```

三电布局不能机械地固定清流。应根据贸易站数量、目标配方和组合总效率，在赤金专用组与通用制造组之间选择。153 只有一个贸易站，清流仅提供 20% 赤金效率，因此冬时 + 温蒂 + 森蚺的 130% 通用组优于清流组的 120%。

### 3.2 成员角色

| 角色 | 干员 | 设施 | 最低练度 | 说明 |
|------|------|------|----------|------|
| 自动化核心 | 温蒂 | 制造 | E2 | 仿生海龙；通用制造效率 |
| 赤金专用候选 | 清流 | 制造 | E1 | 每贸易站 +20%，仅赤金 |
| 通用首选 | 森蚺 | 制造或中枢 | E2 | 三电布局优先制造；252/342有Lancet-2时强制中枢，缺Lancet-2时回制造 |
| 通用候选 | 冬时 | 制造 | 解锁流程优化的练度 | 按房间人数提供通用效率 |
| 通用候选 | 异客 / 掠风 | 制造 | 解锁自动化 α 的练度 | 二者同级，每有效电站 +5% |
| 虚拟电源 | 承曦格雷伊 | 发电站 | E2 | 条件满足时 VirtualPower +1 |
| 两电布局 producer | 森蚺 | 中枢 | E2 | 与 Lancet-2 联动，VirtualPower +2 |
| 两电布局挂件 | Lancet-2 | 发电站 | 0 | 中枢森蚺联动条件 |

### 3.3 243 不使用中枢森蚺

243 有三物理电站，森蚺留在制造站可直接提供每有效电站 10% 的通用制造效率。将森蚺移入中枢并占用 Lancet-2 发电位，会损失制造第三人和其他中枢/发电收益，因此默认不采用该分支。

### 3.4 252/342 的中枢森蚺分支

252/342 只有两物理电站：

- 同时拥有森蚺与Lancet-2时，强制森蚺进中枢、Lancet-2 进发电站，VirtualPower +2。
- 森蚺进入中枢后，禁止同时进入制造站。
- 同时拥有承曦格雷伊时，再安排其进另一发电站，额外 VirtualPower +1。
- 无承曦格雷伊时不关闭体系，只按较低有效电站数结算。
- 默认两人赤金站为清流 + 温蒂；无温蒂 E2 时不启用该自动化组合。
- Lancet-2 必须0心情红脸上班；该组下班时转入加工站，不进宿舍。
- 三级三人站中森蚺已进中枢时，第三位按冬时 ＞ 异客 = 掠风选择。
- 有森蚺但没有Lancet-2时，不走中枢联动，森蚺回制造站。

## 4. 班次与房间约束

- 自动化组合工作两个班次、休息一个班次。
- 第一班默认是主力班；第二个工作班可以是第二班，也可以是第三班，只要符合正常三队换班逻辑。
- 制造组与所有为该组提供虚拟电站的 producer 必须绑定到同一队、同上同下。
- 三电布局绑定制造三人 + 承曦格雷伊（若使用）。
- 252/342 绑定制造成员 + 中枢森蚺 + Lancet-2 + 承曦格雷伊（若使用）。
- Lancet-2 随该组停止发电工作时进入加工站，不进入宿舍；不能套用普通休息队宿舍分配。
- 不固定队名、房间 id 或休息班次编号。
- 清流只允许进入赤金配方；通用制造组可进入合法制造配方。

## 7. 关闭与降级语义

1. 无温蒂 E2：不形成以仿生海龙为核心的自动化组，退回普通制造组合。
2. 无清流或清流未到 E1：不关闭，不能使用清流赤金分支，改评估通用制造组合。
3. 无承曦格雷伊：不关闭，按实际有效电站数降低效率。
4. 有效电站数量没有关闭阈值；不得再使用“至少4”或“低于3关闭”的旧规则。
5. 三电布局的完整自动化制造组固定三人；缺合法第三人时不声称形成满配组，但体系也不整体关闭。
6. 两电布局允许两人制造站；有森蚺时必须切换到中枢森蚺 + Lancet-2 分支。
7. 无赤金线时不关闭，可使用不含清流的通用制造组合。
8. 两电布局的清流 + 温蒂组合缺温蒂 E2时不启用。
9. 三电布局缺森蚺时，以温蒂为核心，从冬时 ＞ 异客 = 掠风中选两名同伴；凑不满三名合法成员时，整房改走普通制造组合。
10. 两电布局有森蚺但没有Lancet-2时，森蚺回制造站，不强行占用中枢。
11. 两电布局清流 + 温蒂两人赤金站在2、3、4、5有效电站时均可正常启用。
12. 两电两人通用站按实际效率动态排序；三级三人通用站使用 §5.5 的固定锚点。
13. 有森蚺但无Lancet-2时，森蚺固定回三级制造站，并使用 §5.5 的赤金/通用锚点。

## 9. 当前实现边界（2026-07-15）

### 9.1 声明式选型与落位

- `data/orchestration_rules.json` 声明二电 / 三电有限 alternatives；通用规则编译器按实际电站数、配方、练度和房间容量选择，不读取布局名或房号。
- 温蒂 E2 是每条 alternative 的独立 required role；缺温蒂时体系关闭，不能靠清流或森蚺形成伪降级。
- 二电布局声明中枢森蚺 + 发电站 Lancet-2；三电布局声明制造或中枢分支，并由同一角色落位器解析到具体房间。
- `operator_available` / `operator_unavailable` gate 保证 alternatives 的互斥与 fallback 次序，不在 executor 写成员姓名分支。
- 选中的制造成员、producer、`work_mood`、`rest_facility` 和 `shift_bind` 都进入统一 `AssignmentPlan`；execute / fill / rotation / export 只消费计划。

### 9.2 机制、轮换与导出

- 清流仍只在赤金配方生效；温蒂、森蚺、冬时、异客/掠风和承曦格雷伊继续由正式 EffectAtom / solver 结算，规则层不复制公式。
- `AssignedOperator.work_mood=0` 由核心 plan 保留并传入发电 solver；Lancet-2 按 0 心情结算时仍计入森蚺 producer workforce，且不阻塞承曦格雷伊晨曦状态。
- MAA JSON 不编码 `AssignedOperator.work_mood`，也不能把 Lancet-2 自动设置为 0 心情；调用排班前，账号必须实际将 Lancet-2 维持在红脸状态。MAA 只导出成员、设施、配方，并通过 `rest_facility=workshop` 保留休息班进入加工站且不进宿舍的去向。
- 全员 `shift_bind` 保证同队上 2 休 1；房间整体切队，不依赖固定第一/三班或房间顺序。
- 自动化组与莱茵生命仍无体系关系；二者只按真实岗位容量与效率竞争。

### 9.3 单一责任边界

- `RuleSpec / AlternativeSpec / RoleSpec` 唯一决定分支、required 成员、配方、工作状态与绑定。
- `AssignmentPlan` 是完整解析结果；制造 fill 只保留全部 anchor 并补房，resolve 只算正式机制，rotation / export 不重新选型。
- 已删除旧自动化专用候选 trace / evaluator 路径，不保留第二事实源。
